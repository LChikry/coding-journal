1. Essential Basic Meta Tags

<title>Your Page Title - Keep it under 60 characters</title>
<meta name="description" content="150-160 character description that appears in search results">
<meta name="keywords" content="keyword1, keyword2, keyword3">

Why important: Title and description are what users see in search results. They directly impact click-through rates.

2. Open Graph (Facebook/LinkedIn)

<meta property="og:title" content="Title for social sharing">
<meta property="og:description" content="Description for social sharing">
<meta property="og:image" content="https://yoursite.com/image.jpg">
<meta property="og:url" content="https://yoursite.com">

Why important: Controls how your site appears when shared on Facebook, LinkedIn, and other platforms.

üí° Image Requirements:
  • Minimum: 600x315px
  • Recommended: 1200x630px
  • Max file size: 8MB
  • Format: JPG, PNG, WebP

3. Twitter Cards

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Title for Twitter">
<meta name="twitter:description" content="Description for Twitter">
<meta name="twitter:image" content="https://yoursite.com/image.jpg">

Card Types:

  • summary - Default card with small image
  • summary_large_image - Large image card (recommended)
  • app - Mobile app promotion
  • player - Video/audio content

4. Technical SEO Meta Tags

<meta name="robots" content="index, follow">
<link rel="canonical" href="https://yoursite.com/page">
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Robots values:

  • index, follow - Allow indexing and following links (default)
  • noindex, nofollow - Don't index page or follow links
  • index, nofollow - Index page but don't follow links

5. Structured Data (JSON-LD)

Structured data helps search engines understand your content better. For a portfolio site, use the "Person" schema:

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Person", "name": "Your Name", "jobTitle": "Front-end Engineer", "url": "https://yoursite.com", "sameAs": ["https://github.com/username", "https://linkedin.com/in/username"] } </script>

6. Performance & Loading

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="dns-prefetch" href="//external-site.com">
<link rel="preload" href="/critical-resource.css" as="style">

Performance hints:

  • preconnect - Establish connection to external domains early
  • dns-prefetch - Resolve DNS for external domains
  • preload - Load critical resources immediately
